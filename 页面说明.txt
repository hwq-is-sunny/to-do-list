【文档用途】
这个文档是给 Cursor 看的，用来帮助它理解我已经做好的 UI 页面。
要求：
1. 不要重做 UI，不要改变整体视觉风格。
2. 不要擅自增加很多新页面。
3. 先按现有页面补功能，让 app 可以运行。
4. 如果有不确定的地方，先按“默认交互假设”理解，不要乱改。

【整体风格要求】
- 深色背景
- 圆角卡片
- 橙色 / 粉色 / 紫色 / 绿色作为重点色
- 风格偏年轻、轻盈、现代
- 底部导航固定保留
- 中间橙色圆按钮是全局重点操作按钮

【默认交互假设】
如果某些地方我没有明确写出来，请先按以下方式理解：
1. 底部中间橙色圆按钮 = 快速新增任务 / 日程
2. 所有任务卡片默认支持点击进入详情或编辑
3. 小圆圈勾选 = 完成任务
4. 三点按钮 = 更多操作（编辑、删除、置顶、归档等）
5. 顶部左右箭头 = 切换日期 / 月份
6. 底部导航从左到右大致是：首页 / 日历或课表 / 新增 / 数据或分析 / 我的或设置
7. 第一版先做基础功能，复杂的 AI、自动同步、智能规划后面再补

-----------------------------------
01-首页（代办总览页）
-----------------------------------
作用：
这是 app 的首页，用来展示当天任务总览、优先任务、完成情况和全部任务列表。

页面组成：
- 顶部问候语
- 月份切换（左右箭头）
- 一周日期条
- Everyday / Weekly 切换
- 今日任务总览卡片
- Priority Task 优先任务卡片
- Completed 完成统计卡片
- In progress 进行中统计卡片
- All Tasks 全部任务列表
- 底部导航栏
- 底部中间橙色新增按钮

点击行为：
- 点击左右箭头：切换月份或日期范围
- 点击某一天：切换到当天任务
- 点击 Everyday / Weekly：切换展示模式
- 点击 Check Now：查看今天任务详情或跳转到任务列表区域
- 点击 Priority Task 中的圆圈：切换任务完成状态
- 点击某个任务项：进入任务详情 / 编辑页
- 点击 All Tasks 中任务右侧更多按钮：弹出编辑、删除、置顶等操作
- 点击底部中间橙色按钮：新增任务
- 点击底部导航：切换到其他模块页面

数据内容：
- 日期
- 今日任务数量
- 紧急任务数量
- 优先任务列表
- 全部任务列表
- 完成数量
- 进行中数量

说明：
- 这个页面是最核心页面，第一版必须先做出来。
- 优先保证：任务展示、勾选完成、切换日期、点击进入详情、新增任务。

02-月历课程页（月历总览页）
-----------------------------------
作用：
用月历的方式展示课程 / 日程分布，并在下方显示某几天的安排明细。

页面组成：
- 返回按钮
- 搜索框
- Done 按钮
- 刷新按钮
- 新增按钮
- 月历区域
- Required Course 必修课程卡片
- Elective 选修课程卡片
- 下方日程明细区域
- 底部导航栏
- 中间橙色按钮

点击行为：
- 点击返回：返回上一页
- 点击搜索框：搜索课程或日程
- 点击 Done：完成当前编辑或选择
- 点击刷新按钮：刷新日程 / 课程数据
- 点击加号按钮：新增课程或新增日程
- 点击月历中的某一天：查看该天对应课程与安排
- 点击下方日程明细项：进入详情页
- 点击底部导航：切换模块
- 点击中间橙色按钮：快速新增

数据内容：
- 月份
- 每天是否有课程或日程
- 必修课程列表
- 选修课程列表
- 某天的详细安排（时间、标题、课程名）

说明：
- 这个页面更偏“月视图 + 当日详情”。
- 第一版可以先做静态月历 + 点击日期切换下方内容。
- 搜索、刷新、课程分类可以后续再逐步补全。

03-注册页（账号创建页）
-----------------------------------
作用：
用于创建账号，也可以作为登录 / 注册流程中的一个页面样式参考。

页面组成：
- 标题：Create your account
- Name 输入框
- Email 输入框
- Password 输入框
- 密码显示/隐藏按钮
- Get Started 按钮
- 底部登录/注册切换文案

点击行为：
- 点击 Name 输入框：输入用户名
- 点击 Email 输入框：输入邮箱
- 点击 Password 输入框：输入密码
- 点击眼睛图标：切换密码可见 / 不可见
- 点击 Get Started：提交注册信息
- 点击底部文案：跳转到登录或注册页面

数据内容：
- 用户名
- 邮箱
- 密码

说明：
- 这个页面不是第一版核心，可以后面再做。
- 如果第一版不做账号系统，可以先保留页面样式，不接真实后端。
- 后面如果要做登录注册，再接接口。

-----------------------------------
作用：
用于管理 app 的主题、提醒、日历显示方式、导入同步等设置。

页面组成：
- 顶部返回按钮

- Theme Colors
- AI Planning
- Tomato Focus
- Schedule Import and Management 区域

- 提醒设置区域
- Calendar view 区域
- 底部导航栏
- 中间橙色按钮

点击行为：
- 点击返回：返回上一页
- 点击 Theme Colors：进入主题颜色设置
- 点击 AI Planning：进入 AI 规划功能页面
- 点击 Tomato Focus：进入番茄专注页面
- 点击某一项带箭头的设置：进入对应二级设置页
- 点击开关：切换开启 / 关闭状态
- 点击底部导航：切换模块
- 点击中间橙色按钮：快速新增

数据内容：
- 是否自动同步日程
- 默认提醒时间
- 默认提醒方式
- 日历显示方式
- 主题设置
- 时间设置

说明：
- 第一版只需要先保留基础设置项即可。
- 可以先实现最简单的几个设置：
  1. 是否提醒
  2. 默认提醒时间
  3. 主题切换（后面可做）
- AI Planning、Tomato Focus 属于后续增强功能。

05-日历页 / 日程时间轴页
-----------------------------------
作用：
以日期 + 时间轴的形式展示当天或某天的全部任务 / 课程安排。

页面组成：
- 页面标题 Calendar
- 月份文字
- 一周日期条
- All Tasks 区域
- 时间轴
- 每个时间段对应的任务卡片
- 右上角更多按钮

点击行为：
- 点击某一天：切换当天日程
- 点击某个任务卡片：查看详情或编辑
- 点击右上角更多按钮：打开更多操作（筛选、切换视图、排序等）
- 上下滑动：浏览全天时间轴任务
- 点击底部导航：切换模块

数据内容：
- 日期
- 时间段
- 任务标题
- 课程名称 / 任务来源
- 地点
- 标签或编号

说明：
- 这个页面核心是“按时间查看任务”。
- 第一版建议先做：
  1. 日期切换
  2. 当天任务按时间排序显示
  3. 点击任务进入详情
- 更复杂的筛选和周视图可以后补。

06-页面之间的跳转关系
-----------------------------------
默认跳转关系如下：
- 首页 -> 点击任务 -> 任务详情 / 编辑页
- 首页 -> 点击橙色按钮 -> 新增任务页
- 首页 -> 点击底部导航日历 -> 日历页
- 首页 -> 点击底部导航我的 -> 设置页
- 日历页 -> 点击任务 -> 任务详情 / 编辑页
- 月历课程页 -> 点击加号 -> 新增课程 / 新增日程
- 设置页 -> 点击设置项 -> 对应二级设置页
- 注册页 -> 点击 Get Started -> 进入 app 首页或登录流程
